5Ô∏è‚É£ Complete Example: What Goes Where
public class BankAccount {
    // Static field - value stored in Heap (Class object)
    private static int totalAccounts = 0;
    
    // Instance fields - metadata in Metaspace, values in Heap (per object)
    private String accountNumber;
    private double balance;
    
    // Constructor - bytecode in Metaspace
    public BankAccount(String accountNumber) {
        this.accountNumber = accountNumber;
        this.balance = 0.0;
        totalAccounts++;
    }
    
    // Method - bytecode in Metaspace
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited: $" + amount);
        }
    }
}

üéØ Key Takeaways

Metaspace Contains:
  • Class structure and metadata
  • Method bytecode and information
  • Field metadata (NOT values)
  • Runtime Constant Pool (per class)
  • Everything loaded ONCE per class
Heap Contains:
  • Object instances (per object)
  • Instance field VALUES
  • Static field VALUES (in Class object)
  • String Pool (string literals)
  • Arrays
üí° Remember:
  • Metaspace = Class-level (structure, definitions, code) - stored ONCE
  • Heap = Object-level (instance data, values) - stored PER OBJECT
  • Runtime Constant Pool is INSIDE Metaspace (part of class metadata)
  • String Pool is INSIDE Heap (Java 7+)
